# FEATURE_ENTRY.prompt.md

Use this before large features.

Read the brain files first.

Then answer:

## Feature
What is being proposed?

## User reason
Why would the user care?

## Product fit
How does this reinforce the thesis?

## Truth check
Does the repo currently have the foundations to support this honestly?

## Contradictions introduced
What stated goals does this strain?

## Complexity cost
What new moving parts, states, dependencies, or support burdens appear?

## Better alternative
Is there a narrower version that captures 80% of the value?

## Decision
- proceed
- proceed with narrower scope
- delay
- reject

If the feature changes long-term direction, create a new ADR.
